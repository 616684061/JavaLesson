## &&、||と&、|の`い

&&、||と&、|において演算子が意味するところは同じですが、その振る舞いには明_な`いがあります。& <br>
&と&は左オペランドと右オペランドのいずれもがtrueの龊trueを返します。このことは左オペランドがfalseの龊悉<br>
右オペランドがtrue、falseにかかわらずfalseを返すとも言えます。&&は左オペランドがfalseの龊嫌谣ペランドをg行せずにfalseを返します。<br>
&は左オペランドがfalseの龊嫌谣ペランドをg行した後、falseを返します。
<br>
<br>
|| と|は左オペランドと右オペランドのいずれかがtrueの龊稀trueを返します。このことは左オペランドがtrueの龊悉嫌谣ペランドがtrue、<br>
falseにかかわらずtrueが返されるとも言えます。||は左オペランドがtrueの龊稀⒂谣ペランドをg行せずにtrueを返します。<br>
|は左オペランドがtrueの龊稀⒂谣ペランドをg行した後trueを返します。<br>
&&、||と&、|の振る舞いの`いは左オペランドのY果のみで全体のg行Y果が判eできるrに右オペランドをg行するかどうかにあります。<br>
&&、||演算子はこれらの特性からショ`トサ`キット（短jする）演算子とも言われています。
<br>
<br>
true、falseの判eのみであればどちらの演算子を使用してもg行Y果に`いはありませんが、<br>
右オペランドに何らかのI理をg行させる龊悉摔险瘠胛瑜い芜`いを意Rする必要があります。<br>
右オペランドに何らかのI理をg行させる龊悉谓Y果の`いを以下に例示します。</span>

【例２】右オペランドで何らかのI理をg行させた龊||と|でY果がなる例を示します。<br>


    public class ExOperator8 {
      public static void main(String[] args) {
        int x1 = 5;
        if (x1 > 0 || (x1 *= 10) > 100) {   //(1)
          System.out.println("x1は"+ x1);
        }

        int x2 = 5;
        if (x2 > 0 | (x2 *= 10) > 100) {   //(2)
          System.out.println("x2は" + x2);
        }
      }
    }

【解h２】
1.||演算子の龊稀⒆螗ペランドがtrueの龊嫌谣ペランドをg行しません。<br>左オペランドがtrueのため、涫x1を10倍すると言う右オペランドのI理が行われずprintln文がg行されます。
2.|演算子の龊稀⒆螗ペランドがtrueの龊悉庥谣ペランドをg行します。<br>右オペランドの涫x1を10倍すると言うI理を行い、println文をg行します。
